# 📈 Telecom X - Predicción de Cancelación (ML Pipeline)

## 📝 Descripción de la Parte 2
En esta segunda etapa del desafío, pasamos del análisis exploratorio a la construcción de una solución de **Inteligencia Predictiva**. El objetivo fue desarrollar un pipeline de Machine Learning robusto capaz de identificar a los clientes con alta probabilidad de abandonar la empresa (*Churn*), permitiendo a Telecom X tomar decisiones basadas en datos para reducir pérdidas.

## 🎯 Objetivos del Desafío
* **Preprocesamiento Avanzado:** Codificación de variables (One-Hot Encoding) y estandarización de datos numéricos.
* **Análisis de Correlación:** Identificación matemática de las variables con mayor impacto en la fuga.
* **Modelado Predictivo:** Entrenamiento y comparación de modelos de clasificación (KNN y Árbol de Decisión).
* **Optimización:** Tratamiento de *Overfitting* mediante técnicas de poda de árboles.
* **Entrega de Insights:** Análisis de importancia de variables y propuesta estratégica.

## 🛠️ Tecnologías y Librerías Utilizadas
* **Scikit-Learn:** Para el modelado, división de datos (`train_test_split`) y métricas de evaluación.
* **Pandas & NumPy:** Manipulación de matrices y datos codificados.
* **Seaborn & Matplotlib:** Generación de matrices de confusión y gráficos de importancia de variables.

## ⚙️ El Pipeline de Machine Learning

### 1. Preparación de Datos
* Se eliminaron identificadores únicos (`customerID`) para evitar ruido.
* Se aplicó **One-Hot Encoding** a las variables categóricas, transformando el texto en datos procesables por los algoritmos.
* Se realizó una división de datos **70% Entrenamiento / 30% Prueba**, utilizando una semilla de aleatoriedad (`random_state=5`) y estratificación para mantener el balance de clases.
* **Estandarización:** Se utilizó `StandardScaler` únicamente en las variables numéricas continuas para asegurar que modelos basados en distancia (como KNN) no se vieran sesgados.

### 2. Entrenamiento y Evaluación de Modelos
Se compararon dos enfoques distintos:
* **K-Nearest Neighbors (KNN):** Un modelo estable que alcanzó un **78% de Accuracy**.
* **Decision Tree:** Inicialmente presentó un sobreajuste severo (99% en Train), el cual fue corregido limitando la profundidad (`max_depth=5`), logrando un modelo final con una **Exactitud del 78.6%** y un **Recall del 54%** para la clase de fuga.

## 📊 Principales Hallazgos (Insights de ML)
A través de la métrica `feature_importances_`, identificamos que tres factores explican la mayoría de las cancelaciones:
1.  **Contrato Mes a Mes:** El predictor de fuga más potente (aprox. 50% de peso).
2.  **Uso de Fibra Óptica:** Los clientes con este servicio presentan una mayor tendencia a abandonar.
3.  **Baja Permanencia (Tenure):** El riesgo de fuga es crítico durante el primer año de contrato.



## 📣 Conclusiones Estratégicas
Para reducir el Churn en Telecom X, se recomienda:
* **Campañas de Conversión:** Ofrecer incentivos para migrar clientes de planes mensuales a contratos anuales.
* **Auditoría Técnica:** Investigar la satisfacción del cliente específicamente en el servicio de Fibra Óptica.
* **Fidelización Temprana:** Implementar beneficios que se activen al cumplir 6 meses de antigüedad para superar la barrera crítica de fuga.
